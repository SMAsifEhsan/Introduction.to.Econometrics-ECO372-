R² says how much of the variation in test scores the regression accounts for. The total variation splits exactly into an explained part and a residual part: TSS = ESS + RSS, and R² = ESS / TSS. Change the scenario and watch the explained (green) share of the bar grow or shrink.

Same line, same data — only the spread changes. Tighter clouds leave a smaller residual (orange) share, so R² rises toward 1; a flat cloud explains nothing, so R² ≈ 0.

Is STR really related to test scores, or could the estimated slope just be sampling noise? We test H₀: β₁ = 0 against H₁: β₁ ≠ 0. The t-statistic counts how many standard errors β̂₁ sits from 0; if it lands in a shaded tail we reject H₀. Shrink the sample or draw a new one and watch the decision flip.

As n grows the standard error shrinks, so |t| grows and even a fixed slope eventually becomes significant — a reminder that “statistically significant” is not the same as “large.”

Data: simulated districts with STR ~ Normal(19.6, 2.3) and Test score = β₀ + β₁·STR + noise, so the data-generating process is known and every quantity can be seen exactly. Each scenario uses a fixed seed, so the picture is identical on every load.

Tests use homoskedastic standard errors and the exact t-distribution with n − 2 degrees of freedom, which approaches the standard normal as n grows. Single self-contained file — runs offline.
